# ProtestTracker

ProtestTracker is an independent, automatically updated archive of protest movements. It pairs hand-curated context (timelines, background, legal proceedings, sources) with a news aggregator that refreshes each movement's recent coverage and computes an activity status on a schedule. The design language is deliberately restrained: an ink-and-paper documentary look meant to read like a printed record rather than a dashboard. The site is a static Next.js export, so there is no server or database. All content ships as JSON and static HTML, and the whole thing is rebuilt and redeployed by CI.

## Tech stack and local development

- **Next.js 16** (pages router) with `output: 'export'` for a fully static build.
- **React 19**, **TypeScript 7** (the native compiler), **Node 20**.
- Data as flat JSON files under `public/data/`.
- Hosted on **GitHub Pages** at [protesttracker.net](https://protesttracker.net).

```bash
npm install        # install dependencies
npm run dev        # start the dev server at http://localhost:3000
npm run typecheck  # type-check with the native TypeScript 7 compiler (tsc)
npm run build      # produce the static export in ./out
```

The build output in `./out` is a self-contained static site. `next.config.js` sets `output: 'export'`, `images.unoptimized: true`, and `trailingSlash: true`.

### A note on TypeScript 7

The project uses TypeScript 7, the native (Go) compiler. It has no JavaScript compiler API, so Next's in-build "Running TypeScript" pass cannot load it, and `@typescript-eslint` does not yet support it. Accordingly:

- `next.config.js` sets `typescript.ignoreBuildErrors: true` to skip only Next's in-build type pass. The native `tsc` binary is the source of truth for types: run `npm run typecheck` locally and in CI (it runs before the build).
- ESLint via `eslint-config-next` is not wired into the build (Next 16 no longer lints during builds); enable `npm run lint` once `@typescript-eslint` supports TS 7.

## Adding a movement

A movement is identified by a stable `id` (the folder name under `public/data/`). Adding one is three steps plus one rebuild.

### 1. Add an entry to `scripts/movements.config.json`

The config is a JSON object keyed by the movement `id`:

```json
{
  "example-movement": {
    "name": "Example Movement",
    "region": "Asia",
    "location": "India · New Delhi",
    "year": 2026,
    "keywords": ["Example Movement", "example protest"],
    "feeds": [
      "https://news.google.com/rss/search?q=%22Example%20Movement%22&hl=en-IN&gl=IN&ceid=IN:en"
    ],
    "manualStatus": null
  }
}
```

- `keywords` are used for matching and search.
- `feeds` is a list of RSS URLs the aggregator fetches (see the Google News RSS caveat below).
- `manualStatus` overrides the computed status when set to one of `Active`, `Escalating`, `Quiet`, `Dormant`, or `Concluded`; leave it `null` to let the aggregator decide.
- For an existing movement, the aggregator preserves the curated `name`, `region`, `location`, `year`, `description`, and `logged` already present in `movements.json`; config values are used only for brand-new ids.

### 2. Create `public/data/{id}/` with the curated files

Create the folder and add the four hand-curated files. Their shapes are documented under [Editing curated data](#editing-curated-data). At minimum, create `timeline.json`, `background.json`, `legal.json`, and `sources.json`. You do NOT need to create `articles.json`: the aggregator creates and refreshes it.

### 3. Add the movement to `public/data/movements.json`

Add the movement to the index at `public/data/movements.json`, or let the aggregation script generate and refresh the entry for you. Either way, note the following:

- Adding a **new** `id` requires one rebuild so its `/movements/{id}/` page is statically generated. `getStaticPaths` reads `movements.json` at build time, so a page only exists for ids present in `movements.json` when the build runs.
- `articles.json` will be created and refreshed by the aggregation script; you do not author it by hand.

## Editing curated data

Four files per movement are hand-edited and are **never** overwritten by automation. `scripts/aggregate.mjs` only rewrites `articles.json` (per movement) and `movements.json` (the index); it never touches `timeline`, `background`, `legal`, or `sources`.

Copy rule: no em-dashes anywhere in prose or curated text. Use commas, colons, or periods instead.

### `timeline.json`

An array of events.

```json
[
  { "date": "2026-01-15", "title": "First march", "body": "Thousands gathered in the capital." }
]
```

### `background.json`

An array of content blocks. `type` is `"h"` for a heading or `"p"` for a paragraph.

```json
[
  { "type": "h", "text": "Origins" },
  { "type": "p", "text": "The movement began as a response to proposed legislation." }
]
```

### `legal.json`

An array of legal cases.

```json
[
  {
    "name": "State v. Organizers",
    "court": "High Court",
    "status": "Quiet",
    "label": "Pending",
    "summary": "Charges filed against several organizers.",
    "updated": "2026-03-02"
  }
]
```

### `sources.json`

An array of sources.

```json
[
  { "name": "Independent Media Trust", "type": "news", "note": "Regional coverage." }
]
```

## How status is computed

Status is computed entirely by `scripts/aggregate.mjs` from a movement's articles. It is never hand-set in the UI: the value shown comes from `movements.json`, which the aggregator produces. The rules:

- **Active**: at least one article in the last 7 days.
- **Quiet**: most recent article is 8 to 30 days old.
- **Dormant**: most recent article is more than 30 days old.
- **Escalating**: the count of articles in the last 48 hours is at least 3x the trailing 14-day daily average.
- **manualStatus override**: if a movement sets `manualStatus` in `scripts/movements.config.json`, that value overrides the computed status.

## Running the aggregator

```bash
npm run aggregate
```

This runs `node scripts/aggregate.mjs`, which fetches the configured RSS feeds, rewrites each movement's `articles.json`, refreshes `movements.json` (including the computed status), and leaves the curated files untouched. In CI the aggregator runs every 2 hours (cron) and on every push to `main`, and can be triggered manually via `workflow_dispatch`.

To refresh only specific movements, pass their ids (other movements keep their existing entries):

```bash
npm run aggregate -- india-cjp nigeria-fuel
```

The `legal.json` `status` field reuses the same five statuses as movements (`Active`, `Escalating`, `Quiet`, `Dormant`, `Concluded`) so legal cases render with the shared status colors.

## Google News RSS caveat

Google News search RSS URLs are finicky about encoding, and empty results usually trace back to the query string or the locale params:

- Encode spaces as `%20` and quotes as `%22`.
- The `hl` (language), `gl` (country), and `ceid` (country:language) params matter and change which results come back.
- Before relying on a feed, verify it returns items: open the URL in a browser or `curl` it and confirm there are `<item>` entries. If it is empty, adjust the query encoding or the `hl`/`gl`/`ceid` locale params.

Example working URL format:

```
https://news.google.com/rss/search?q=%22Cockroach%20Janta%20Party%22&hl=en-IN&gl=IN&ceid=IN:en
```

```bash
# Quick check that a feed returns items
curl -s "https://news.google.com/rss/search?q=%22Cockroach%20Janta%20Party%22&hl=en-IN&gl=IN&ceid=IN:en" | grep -c "<item>"
```

## DNS and custom domain setup

The custom domain lives in `public/CNAME`, which contains `protesttracker.net`.

In the GitHub repository under **Settings > Pages**:

1. Set **Source** to **GitHub Actions**.
2. Set the **Custom domain** to `protesttracker.net`.
3. Enable **Enforce HTTPS**.

DNS configuration:

- For the apex domain `protesttracker.net`, create an APEX/ALIAS record, or four `A` records pointing to the GitHub Pages IPs:

  ```
  185.199.108.153
  185.199.109.153
  185.199.110.153
  185.199.111.153
  ```

- For `www`, create a `CNAME` record pointing to `<user>.github.io`.

The CI build emits a `.nojekyll` file into `./out` (via `touch out/.nojekyll`) so GitHub Pages does not run Jekyll and strip the `_next` directory.

## Neutrality

This archive is independent and is not affiliated with any political party, campaign, government, or advocacy group. It aims to document movements factually and without endorsement. Curated copy is written to describe events, not to take sides.
